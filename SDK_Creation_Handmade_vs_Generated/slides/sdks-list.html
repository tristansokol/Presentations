And out pops our SDKs. We have six first party SDKs, each one gets its own repo on github, and we also have another repo that houses our specification, our templates, and all our configuration in a separate github repo. And this was a pretty happy state. We would have upstream teams make an API change or release a new feature, then we would regenerate our specification and then manually generate each of the SDKs.
